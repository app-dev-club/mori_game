# レート計算アルゴリズム

本ゲームのレーティングは **OpenSkill**（Weng-Lin の Plackett-Luce モデル）を用いています。  
各プレイヤーの実力を正規分布で表し、試合結果に応じて更新します。

## 基本コンセプト

| 記号 | 意味 |
|------|------|
| **μ (mu)** | スキルの平均（高いほど強い） |
| **σ (sigma)** | 不確実性（大きいほど試合数が少なく、レートの信頼度が低い） |
| **表示レート** | 画面上に表示する整数値（ランキング・ロビー用） |

プレイヤーごとに `(μ, σ)` を Firebase の `ratings/{playerId}` に保存します。

## 初期値

| 項目 | 値 |
|------|-----|
| μ | 25.0 |
| σ | 25/3 ≈ 8.33 |
| 表示レート | **1500** |

表示レートは次式で求めます。

```
ordinal = μ − 3σ
表示レート = round(ordinal + 1500)
```

初期状態では `ordinal = 0` のため、表示レートは 1500 になります。

## 更新タイミング

- **シリーズ対戦がすべて終了したとき**（例: 3 戦セットの最終戦後）に 1 回だけ更新
- 同じルームへの二重適用は `seriesRatingApplied` フラグで防止

## 更新の流れ

1. **順位決定**  
   シリーズ最終時点の累計ポイント（`playerPoints`）を降順に並べ、順位を付ける。  
   同点のプレイヤーは同順位とする。

2. **OpenSkill で μ・σ を更新**  
   - 各プレイヤーを 1 人チームとして扱う（FFA 形式）
   - 順位を `ranks` として Plackett-Luce モデルに入力
   - 勝者は μ が上がり、敗者は μ が下がる
   - 試合を重ねるほど σ は小さくなり、レートの確信度が上がる

3. **表示レートの増減**  
   更新前後の表示レートの差を `ratingDelta` として記録する。

```
ratingDelta = 表示レート(更新後) − 表示レート(更新前)
```

## σ の見方

- **σ が大きい** … まだ試合数が少なく、真の実力が不確か
- **σ が小さい** … 多くの試合で μ が安定しており、レートの信頼度が高い

ロビー画面では `レート 1500 · σ 8.33` のように σ も表示します。

## 旧データ（ELO のみ）の移行

μ・σ が未保存の既存プレイヤーは、保存されている `rating` から次式で μ を復元します。

```
ordinal = rating − 1500
μ = ordinal + 3 × (25/3)
σ = 25/3（デフォルト）
```

| ファイル | 役割 |
|----------|------|
| `lib/logic/open_skill.dart` | Plackett-Luce コア（μ・σ の更新） |
